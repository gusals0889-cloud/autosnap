기획 (Planning)

문제 인식

교육, 회의, 협업, 고객 지원, 개발 문서 작성 등 다양한 상황에서 사용자는 화면의 일부를 빠르게 캡처하고, 그 위에 화살표나 텍스트를 덧붙여 즉시 전달할 필요가 있다. 하지만 기존 도구는 설치 과정이 번거롭거나, 기능이 복잡하거나, 캡처 후 편집 단계가 길어 순간적인 전달에 적합하지 않은 경우가 많다.

대상 사용자

  • 사무직 사용자: 회의 화면, 보고용 자료, 메신저 공유용 이미지가 필요한 경우
  • 개발자: 오류 화면, 로그, UI 버그를 빠르게 캡처해 공유해야 하는 경우
  • 강사/학생: 수업 자료, 발표 화면, 설명용 이미지를 만드는 경우
  • 고객 지원/운영 담당자: 문제 재현 화면을 빠르게 기록해야 하는 경우

개발 동기

  • 설치 없이 즉시 실행 가능한 경량 캡처 도구에 대한 필요
  • 캡처와 주석을 하나의 흐름으로 연결해 작업 시간을 줄이고자 하는 목표
  • Windows 환경에서 시스템 트레이 기반으로 상시 대기하는 편의성 제공
  • 단일 실행 파일 배포를 통해 사용 장벽을 낮추고자 함

구성물 (Artifacts)

소스 파일 및 디렉터리 구조

  • src/ - 핵심 애플리케이션 코드
    • main.py - 프로그램 진입점, 트레이 아이콘 초기화, 이벤트 루프 관리
    • selection.py - 반투명 전체화면 레이어 및 드래그 기반 영역 선택
    • annotate.py - 화살표, 텍스트, 간단한 편집 UI 처리
    • utils.py - 이미지 저장, 클립보드 복사, 공통 유틸리티
    • config.py - 기본 설정, 테마, 단축 옵션 값 관리
  • assets/ - 아이콘, 샘플 이미지, UI 리소스
  • tests/ - 기능 검증용 테스트 코드
  • docs/ - README, 사용법, 변경 내역

CI 워크플로우

  • .github/workflows/build.yml - GitHub Actions 기반 자동 빌드
  • 코드 푸시 또는 태그 생성 시 자동 실행
  • PyInstaller를 사용해 autosnap.exe 단일 파일 패키징
  • 빌드 산출물 업로드 및 Release 생성 보조

Release Asset

  • autosnap.exe - Windows용 단일 실행 파일
  • 릴리스 노트, 사용 가이드, 스크린샷 이미지
  • 향후 포터블 ZIP 패키지 지원 가능성

실현 방법 (Implementation Details)

아키텍처 다이어그램

위 이미지는 아키텍처 설명용 플레이스홀더입니다.

흐름도 설명

  1. 프로그램 시작 후 시스템 트레이 아이콘을 생성한다.
  2. 사용자가 트레이 메뉴에서 캡처 시작을 선택한다.
  3. SelectionWindow가 화면 전체를 덮는 투명 레이어를 표시한다.
  4. 사용자가 마우스로 영역을 드래그해 선택한다.
  5. 선택 영역이 확정되면 AnnotateWindow가 열리고 주석 도구가 활성화된다.
  6. 편집 완료 후 이미지를 저장하거나 클립보드로 복사한다.

결과 (Result)

사용 방법

  1. 실행 파일 autosnap.exe를 더블클릭한다.
  2. 시스템 트레이 아이콘에서 캡처 시작을 선택한다.
  3. 마우스로 원하는 화면 영역을 드래그한다.
  4. 주석 도구로 화살표나 텍스트를 추가한다.
  5. 완료 후 PNG로 저장하거나 클립보드에 복사한다.

한계점 (Limitations)

기술적 제약

  • Windows 중심 구현으로 다른 운영체제 지원이 제한적이다.
  • 전역 핫키, 멀티 모니터 정밀 스케일링은 향후 보완이 필요하다.
  • PDF, SVG 등 고급 출력 형식은 아직 지원하지 않는다.
  • 고급 편집 기능은 전문 그래픽 도구에 비해 제한적이다.

테스트 커버리지

  • 핵심 로직에 대한 단위 테스트는 일부 구성되어 있다.
  • GUI 상호작용에 대한 자동화 테스트는 충분하지 않다.
  • 실제 배포 환경에서의 호환성 검증은 추가 필요하다.
